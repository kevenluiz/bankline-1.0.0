# App Android de Extrato Bancário  

## About Me  

I'm Keven Luiz! I'm here to contribute my knowledge on this Android application using Kotlin.  

This project is a banking application developed for Android, utilizing Kotlin, Coroutines, and LiveData. The main objective is to facilitate the viewing and management of bank statements.

## Technologies and Concepts Covered  

- **Kotlin**: Primary language for development, leveraging its modern and concise features.  

- **Coroutines**: Used to simplify asynchronous calls, allowing for smooth operations without blocking the user interface.  

- **LiveData**: Facilitates reactive communication between the UI layer and the ViewModel, ensuring that changes in data are automatically reflected in the interface.  

- **Object-Oriented Programming (OOP)**: Application of OOP concepts from Java, organizing the code into classes and objects for a modular and reusable structure.  

- **Material Design**: Implementation of an intuitive and attractive interface, following Google's UI/UX guidelines.  

- **API Consumption (REST)**: Integration with REST APIs to retrieve relevant user and transaction data effectively.  

- **PostgreSQL Database**: Used as the back-end solution for robust data persistence and retrieval.  

## Code Structure and Practices  

The code is structured following architectural patterns such as **MVVM (Model-View-ViewModel)** and **MVP (Model-View-Presenter)**, ensuring a clear separation of responsibilities. In addition, **Clean Code** practices have been adopted to maintain readable and structured code, along with the principles of **Clean Architecture** for better scalability and maintainability. Various design patterns have been employed to optimize the application's structure.
